# Bifrost 第一阶段详细开发清单

> **执行要求：**
> 1. 所有任务使用复选框跟踪。
> 2. 任务完成后必须在同一行末尾追加完成时间。
> 3. 完成时间统一使用 `Asia/Shanghai` 时区，格式为 `YYYY-MM-DD HH:mm CST`。
> 4. 示例：`- [x] 完成 pnpm workspace 初始化（完成时间：2026-04-17 14:30 CST）`
> 5. 未完成任务保持 `- [ ]`，不要预填时间。

**目标：** 交付 Bifrost 第一阶段最小可用闭环，支持后台配置、客户端登录、设备信任、授权访问私有 Web 服务、网关转发、审计与多容器联调。

**架构：** 采用单仓 Monorepo 管理 `desktop`、`admin`、`gateway` 和共享包；Go 服务端承载鉴权、设备信任、服务目录、策略和反向代理；Electron 客户端保持小卡片式轻量入口；本地通过 Docker Compose 模拟多个私有上游服务。

**技术栈：** Go 1.26.2、PostgreSQL 18.x、Electron 41.2.1、React 19.2.5、TypeScript 6.0.3、Vite 8.0.8、Tailwind CSS v4、shadcn/ui、pnpm、Turborepo、Vitest、Playwright。

---

## 使用规则

### 清单维护规则

- 每完成一个任务，立即勾选并记录完成时间。
- 若任务被拆分，不在原项下写子列表，直接新增新的同级复选项。
- 若任务废弃，保留原项并在后面加说明，例如 `（废弃，原因：...）`。
- 任何变更都不得删除历史完成时间。

### 完成定义

只有同时满足以下条件，复选框才能勾选：

- 代码或文档已经实际落地
- 相关检查或测试已执行
- 结果符合该任务的验收标准
