# 后台管理端 UI 规范

## 1. 设计定位

后台是安全控制台，核心目标是高效、清晰、可追溯。它不应像营销首页，也不应像普通内容站点。

后台应优先支持：

- 高频列表管理
- 快速筛选
- 批量判断
- 详情抽屉
- 审计追踪

## 2. 主布局

推荐：

```text
┌────────────────────────────────────────────┐
│ Top Bar                                    │
├──────────────┬─────────────────────────────┤
│ Sidebar      │ Page Content                │
│              │                             │
└──────────────┴─────────────────────────────┘
```

尺寸：

- Sidebar：`232px`
- Top Bar：`52px`
- 页面左右 padding：`24px`
- 页面上下 padding：`20px`

## 3. 页面结构模板

管理页面统一结构：

```text
Page Header
Filter Bar
Action Bar
Data Table
Pagination
Detail Drawer
```

规则：

- 页面标题不超过 `16px`
- 筛选器优先横向排列
- 表格是主要承载体
- 详情信息优先在右侧 Drawer 展示

## 4. 表格规范

表格是后台核心组件。

尺寸：

- 表头高度：`36px`
- 默认行高：`36px`
- 紧凑行高：`32px`

样式：

- 边框 `1px`
- 行分割线使用 `border-soft`
- hover 行使用轻背景
- 不使用厚重阴影

列设计：

- 第一列通常是主要对象名称
- 状态列使用 Badge
- 时间列使用统一格式
- 操作列靠右

## 5. 筛选器规范

筛选器包括：

- 搜索框
- 状态选择
- 角色选择
- 服务选择
- 时间范围

尺寸：

- 输入框高度 `32px`
- 筛选器间距 `8px`
- 筛选区与表格间距 `12px`

规则：

- 不把所有筛选项都做成大卡片
- 高级筛选可以折叠
- 清空筛选入口必须明显

## 6. 操作按钮规范

主操作：

- 页面右上角
- 使用 `btn-md`
- 例如“创建用户”“创建服务”

行内操作：

- 使用 `btn-xs` 或图标按钮
- 危险操作不直接裸露删除，优先放更多菜单或二次确认

危险操作：

- 使用 danger 语义色
- 必须二次确认
- 确认弹窗中说明影响范围

## 7. Drawer 规范

详情与编辑优先使用 Drawer。

宽度：

- 默认：`480px`
- 复杂表单：`640px`
- 审计详情：`720px`

用途：

- 用户详情
- 设备详情
- 服务详情
- 审计详情

规则：

- Drawer 标题使用 `16px`
- 分组标题使用 `14px`
- 字段标签使用 `12px`

## 8. Modal 规范

Modal 只用于：

- 高风险确认
- 小型创建表单
- 不适合 Drawer 的短流程

宽度：

- `480px`
- `640px`
- `800px`

## 9. 状态 Badge

常见状态：

- Enabled
- Disabled
- Trusted
- Denied
- Allowed
- Archived

Badge 样式：

- 高度 `22px`
- 圆角 `6px`
- 字号 `12px`
- 背景使用语义浅色
- 文字使用语义主色

## 10. 审计页面规范

审计页面应支持：

- 时间范围
- 事件类型
- 操作者
- 目标对象
- 服务
- 结果

审计详情必须展示：

- requestId
- 事件类型
- 操作者
- 设备
- 来源 IP
- 错误码
- 时间
- 结构化 details

## 11. 空状态与错误状态

空状态必须给出下一步动作。

示例：

- “暂无服务，创建第一个私有服务”
- “暂无审计记录，调整筛选条件后重试”

错误状态必须展示：

- 用户可读错误
- requestId
- 重试或返回操作

## 12. 响应式策略

后台主要面向桌面宽屏，但需要基本适配窄屏。

断点策略：

- 小于 `1024px`：侧栏可折叠
- 小于 `768px`：后台不保证完整管理体验，但页面不能崩坏

## 13. 验收标准

- 用户、角色、设备、服务页面结构一致
- 表格、筛选、Drawer 风格统一
- 危险操作都有确认
- 审计事件可快速定位 requestId
- Light / Dark 下状态色都可读
