# 客户端本地存储与设备信任

## 1. 目标

本文档定义桌面客户端如何保存本地敏感信息、如何建立设备信任、如何处理设备吊销。

核心原则：

- 私钥不明文落盘
- 刷新令牌不保存在普通文件
- 设备身份可验证、可吊销
- Renderer 不能直接读取敏感材料

## 2. 本地存储分类

客户端本地数据分为三类：

## 2.1 高敏感数据

包括：

- 设备私钥
- 刷新令牌
- 会话密钥材料

要求：

- 必须存入系统安全存储
- 不允许放入普通 JSON、localStorage、IndexedDB
- 不允许被 Renderer 直接读取

## 2.2 中敏感数据

包括：

- 服务端地址
- 设备 ID
- 当前登录用户摘要
- 最近登录时间

要求：

- 可以保存在应用配置中
- 不得包含令牌或私钥

## 2.3 低敏感数据

包括：

- 主题偏好
- 窗口尺寸
- 最近使用的服务分组

要求：

- 可普通持久化
- 不得混入敏感字段

## 3. 系统安全存储

不同平台建议：

- macOS：Keychain
- Windows：Credential Manager 或 DPAPI
- Linux：Secret Service

抽象层职责：

- 保存设备私钥
- 保存刷新令牌
- 删除会话材料
- 检查是否存在设备身份

## 4. 设备密钥

客户端首次注册时生成设备密钥对。

建议：

- 签名算法采用 `Ed25519`
- 服务端保存公钥和公钥指纹
- 客户端保存私钥

设备 ID 与公钥指纹都可以展示给用户和管理员，但私钥不能展示。

## 5. 设备注册流程

1. 客户端生成设备密钥对
2. 用户输入账号密码登录
3. 客户端提交设备公钥、设备名称、系统信息
4. 服务端创建设备记录
5. 服务端返回设备 ID
6. 客户端保存设备 ID
7. 服务端记录审计事件

## 6. 设备挑战流程

1. 客户端发起登录或刷新
2. 服务端返回挑战值
3. 客户端使用设备私钥签名
4. 服务端使用已登记公钥验签
5. 验签通过后继续会话流程

挑战值要求：

- 一次性
- 短期有效
- 与用户、设备、会话请求绑定

## 7. 设备吊销

管理员禁用设备后：

- 设备不能继续刷新会话
- 已有会话应被吊销
- 后续服务访问被拒绝
- 审计记录设备禁用与拒绝访问事件

客户端表现：

- 清理本地会话
- 保留设备 ID 以展示状态
- 提示“当前设备已被禁用”

## 8. Renderer 安全边界

Renderer 禁止：

- 读取私钥
- 读取刷新令牌明文
- 调用任意系统命令
- 读取任意本地文件

Renderer 只允许通过 Preload 进行受控调用。

## 9. 会话清理

以下场景必须清理本地会话：

- 用户主动退出
- 刷新令牌失效
- 会话被服务端吊销
- 设备被禁用
- 用户被禁用

清理内容：

- 访问令牌
- 刷新令牌
- 当前用户会话状态

是否删除设备私钥应区分：

- 退出登录不删除设备私钥
- 用户主动重置设备身份时才删除设备私钥

## 10. 本地诊断信息

客户端可保存诊断日志，但必须脱敏。

禁止记录：

- 密码
- 访问令牌
- 刷新令牌
- 设备私钥
- 完整 Cookie

允许记录：

- requestId
- 错误码
- 时间戳
- 客户端版本
- 设备状态摘要

## 11. 安全红线

- 不允许私钥写入普通配置文件
- 不允许把刷新令牌放入 localStorage
- 不允许 Renderer 直接访问 Node.js 安全能力
- 不允许日志输出敏感令牌
