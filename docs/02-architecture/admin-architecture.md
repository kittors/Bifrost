# 后台架构与页面流转

## 1. 定位

Web 管理后台是 Bifrost 的安全控制台，用于配置用户、角色、设备、私有服务、授权策略和审计查询。

后台的核心目标是：

- 配置清晰
- 操作可审计
- 风险可识别
- 高频管理效率高

## 2. 技术形态

后台采用：

- `React`
- `TypeScript`
- `Vite`
- `Tailwind CSS v4`
- `HeroUI`
- `TanStack Router`
- `TanStack Query`
- `react-hook-form`
- `Zod`

后台与客户端共享：

- 固定版本 `@heroui/react@3.0.3`
- `packages/design-tokens`
- `packages/contracts`
- `packages/shared-types`
- `packages/shared-utils`

## 3. 页面信息架构

建议页面如下：

```text
admin/
  login
  dashboard
  users
  roles
  devices
  services
  audit-events
  settings
```

## 4. 导航结构

左侧主导航：

- 概览
- 用户
- 角色
- 设备
- 服务
- 审计
- 设置

顶部上下文栏：

- 当前页面标题
- 全局搜索入口，第一阶段可预留
- 当前管理员
- 主题切换
- 退出登录

## 5. 页面设计

## 5.1 登录页

目标：

- 管理员登录后台
- 不提供注册链接

字段：

- 用户名
- 密码

状态：

- 登录中
- 账号密码错误
- 账号禁用
- 服务不可用

## 5.2 概览页

目标：

- 快速判断系统状态和风险

模块：

- 用户总数与禁用数
- 设备总数与禁用数
- 服务总数与停用数
- 今日访问总数
- 今日拒绝次数
- 最近审计事件
- 风险提醒

概览页不做大面积营销式图表，优先工作面板。

## 5.3 用户页

目标：

- 管理用户账号
- 配置角色
- 查看用户设备
- 设置用户级服务覆盖

主要交互：

- 用户列表
- 筛选
- 创建用户
- 编辑用户
- 重置密码
- 禁用/启用用户
- 查看用户详情抽屉

## 5.4 角色页

目标：

- 管理角色
- 配置角色可访问服务

主要交互：

- 角色列表
- 创建角色
- 编辑角色
- 删除角色
- 设置服务授权

## 5.5 设备页

目标：

- 查看与管理已绑定设备
- 禁用可疑设备
- 解绑设备

主要交互：

- 设备列表
- 按用户筛选
- 按状态筛选
- 查看设备详情
- 禁用/启用设备
- 解绑设备

## 5.6 服务页

目标：

- 管理私有 Web 服务目录
- 配置上游地址
- 配置启用状态
- 配置角色授权

主要交互：

- 服务列表
- 创建服务
- 编辑服务
- 启用/禁用服务
- 删除或归档服务
- 查看授权摘要

## 5.7 审计页

目标：

- 查询登录、访问、配置变更、拒绝事件

主要交互：

- 时间范围筛选
- 事件类型筛选
- 操作者筛选
- 服务筛选
- 结果筛选
- 查看详情
- 导出，第一阶段可限制范围

## 5.8 设置页

第一阶段只保留必要设置：

- 会话有效期
- 设备绑定数量上限
- 审计保留策略
- 默认访问策略

不做复杂配置中心。

## 6. 后台状态管理

后台状态分为：

- Server State：用户、角色、设备、服务、审计列表
- Form State：表单编辑状态
- UI State：主题、侧栏、筛选器展开
- Auth State：当前管理员与会话

Server State 统一通过 TanStack Query 管理。

## 7. 表单规范

表单必须：

- 使用统一校验 schema
- 展示字段级错误
- 保存成功后有明确反馈
- 高风险操作必须二次确认

高风险操作包括：

- 禁用用户
- 禁用设备
- 修改服务上游地址
- 禁用服务
- 删除角色
- 导出审计

## 8. 权限显示原则

后台需要清楚展示权限来源：

- 角色授权
- 用户级允许
- 用户级禁止

当用户被禁止访问某服务时，应展示“用户级禁止优先级最高”。

## 9. 审计可见性

所有关键操作成功或失败后，都应能在审计页中查到。

后台页面中高风险操作结果应展示：

- 操作是否成功
- requestId
- 是否已记录审计

## 10. 第一阶段验收标准

- 管理员可登录后台
- 用户、角色、设备、服务可管理
- 角色授权与用户覆盖可配置
- 审计事件可查询
- 所有列表具备分页
- 错误响应统一展示
- 页面符合紧凑清爽设计规范
