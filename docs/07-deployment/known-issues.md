# 已知问题列表

## 第一阶段已知限制

- Desktop Client 是小卡片访问入口，不是 VPN 客户端。
- 客户端不会修改系统代理、DNS 或路由，因此非浏览器类私有协议不在第一阶段范围内。
- 桌面安装包当前先完成三端构建链路，正式发布前仍需补齐代码签名、公证和品牌图标。
- Gateway 第一阶段只代理已配置的 HTTP/HTTPS Web 服务。
- 第一阶段不默认引入 Redis，横向扩展时需要重新评估会话吊销和审计写入压力。
- 自动更新机制尚未接入。

## 运维注意事项

- 默认种子密码只能用于本地或试用环境。
- 生产环境必须设置强随机 `BIFROST_TOKEN_SECRET`。
- 生产数据库必须纳入备份和恢复演练。
- TLS 证书到期前必须有提醒和轮换记录。
